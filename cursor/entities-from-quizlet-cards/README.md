## Методология

- Парсинг: `ответ<TAB>вопрос`; многострочные вопросы игнорируются (берётся только строка с TAB).
- Классификация **только по тексту ответа**, без контекста вопроса.
- При сомнении ответ попадает в `unmapped.json`.
- Составные ответы (`/`, `A и B`), описательные фразы и общие термины — unmapped.

## Ограничения

- Не 100% разметка: «Филадельфия» может быть городом в одной карточке и контекстом в другой.
- Род/падеж не кодируются — только тип сущности.
- Цель — обучающий корпус для расширения `text_parsing_guide.md`.
