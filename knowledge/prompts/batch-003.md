# Генерация карточек (knowledge RAG prompt)

Сгенерируй карточки для сообщений в конце файла. Формат выхода — как в `knowledge/draft/{batch}.json`: массив examples с `message_id`, `id`, `text`, `tags`, `entities`, `cards[{question, answer, logic}]`.

## Core rules

# Правила генерации карточек для quizlet-parser

## Общие принципы
- Мы генерируем флэш-карточки в формате вопрос/ответ.
- Желательная форма вопроса - не в форме вопроса, а с заменой спрашиваемой реалии на ОН/ОНА/ОНО/ТАМ/В ЭТУ ДАТУ/В ЭТОМ ГОДУ/ТАКОЙ/ТАКАЯ/ТАКОЕ/ДЕЛАЛ ЭТО/СДЕЛАЛ ЭТО/СТОЛЬКО (в правильном падеже и правильном роде) итд.
- Входящие данные - это сообщение, которое отправлено в телеграм. Оно может быть чисто текстовым, или содержать ссылку, откуда нужно достать факты для карточек.
- Если сообщение содержит только текст (без ссылки), то:
   - Нужно достать интересные факты именно из текста сообщения и на их основе сгенерировать вопросы/ответы.  
   - Недостающие данные для вопроса/ответа нужно взять из страницы русской википедии. Например, если дана только фамилия, нужно найти её википедию и взять оттуда полное имя.
- Если сообщение содержит и ссылку, и текстовый комментарий 
  - основа вопроса берётся из текстовой части
  - ответ нужно достать из ссылки.
  - дополнительно нужно сходить на страницу по ссылке (почти всегда это будет страница русской википедии), считать весь её текстовый контент, достать интересные факты и на их основе тоже сгенерировать карточки. 
- Если сообщение содержит только ссылку, то идем на страницу и достаем интересные факты.
- Если текст — уже вопрос, то ответ на этот вопрос нужно найти и поместить схожие вопрос/ответ в карточки.

- Одно сообщение текст может давать 1 или более карточек.
- Задаём вопросы по всем реалиям, которые даны в вопросе. По всему, что было записано в entities.
- Вопрос всегда с подсказкой-пустышкой: ОН/ОНИ/ЭТО/ТАМ/ЕГО, либо описательная форма без прямого названия.
- Не надо задавать вопрос "КОГДА", нужно ".... произошло В ЭТУ ДАТУ." или ".... произошло В ЭТОМ ГОДУ."
- Ответ — короткая точная единица (сущность, имя, дата, цитата, термин).
- Если сообщение содержит вопрос, то нужно найти ответ в интернете и на основе этого создать вопрос/ответ.

- Если в вопросе есть персоналии (личности), то их нужно спрашивать в карточках, заменяя на ОН/ОНА.
- Вопрос может быть не идеален по грамматике, но в карточках нужно всё оформлять правильно по грамматике.
- Не нужно задавать вопросы с формой - "... - это ...". Просто пишем определение в вопрос.
- Когда ответ нужно выдать на английском или другом не русском языке, нужно явно объявить это в вопросе.
- Вопрос должен быть понятен чисто по тексту этого вопроса, БЕЗ исходного текста и БЕЗ текстов других карточек.
- Необходимо добавить вопрос к каждой сущности из текста.

## Где находятся примеры

Примеры вопросов/ответов находятся в parsing-examples.json.
Примеры хорошо/плохо сейчас находятся в этом файле (quizlet-rules.md).
Примеры существующих карточек (вопрос/ответ, БЕЗ исходного текста) находятся в директории `/quizlet-modules`. 

Вопросы нужно задавать к реалиям из текста вопроса или текста статьи википедии.
Некоторые из категорий реалий описаны в text_parsing_guide.md


# Запрещено
- Повторять ответ внутри вопроса дословно.
- В одной карточке использовать однокоренные слова в вопросе и ответе ТОЙ ЖЕ КАРТОЧКИ. В разных карточках к тому же тексту использовать однокоренные слова МОЖНО.
- Использовать многословные ответы (>12 слов).


# Примеры

## Пример 1

**Правило:** 
Не задаём вопросы в форме "Кто ...", а заменяем вопросную реалию на слово ОН/ОНА.

**Текст:**
Бродский: Если Евтушенко против, то я за

**Плохо:**
Кто сказал: «Если Евтушенко против, то я за!»?

**Хорошо:**
ОН сказал: «Если Евтушенко против, то я за!»


## Пример 2

**Правило:**
Контекст и часть речи замены должен учитываться. Если заменяется прилагательное, то оно должно спрашиваться как ТАКОЙ/ТАКАЯ/ТАКОЕ (в правильном падеже).

**Текст:**
смерть — костлявая

**Плохо:**
Что такое «смерть»?

**Хорошо:**
Смерть — ТАКАЯ

## Пример 3

**Правило:**
Если ответ надо дать по-английски (или любой другой не русский язык), то нужно об этом явно объявить в вопросе.
Если несколько реалий в ответе, нужно об этом тоже объявить в вопросе.

**Текст:**
firefly, lightning bug — светлячки

**Плохо:**
«светлячки» — это...

**Хорошо:**
Слово «светлячки» на английский можно перевести ЭТИМИ двумя вариантами.

## Пример 4

**Правило:**
Не нужно дробить одну персоналию на вопрос и ответ.

**Плохо:**
Вопрос: Халк — alter ego учёного Брюса…
Ответ: Бэннера

**Хорошо:**
Вопрос: Халк — alter ego ЭТОГО УЧЁНОГО
Ответ: Брюс Бэннер

**Текст:**
Проект «Гутенберг» — публичная цифровая библиотека, основанная Майклом Хартом В ЭТОМ ГОДУ.

**Плохо:**
Вопрос: Крупнейшую волонтёрскую цифровую библиотеку свободных книг называют проектом ЭТИМ.
Ответ: Гутенберг

**Хорошо:**
Вопрос: Крупнейшая волонтёрская цифровая библиотека свободных книг называется ТАК.
Ответ: Проект «Гутенберг»

## Пример 5

**Правило:**
Нельзя ни одну реалию из ответа давать в вопросе к этому ответу, даже если она на другом языке и даже если это однокоренное слово.

**Плохо:**
Вопрос: River в покере — пятая общая карта на столе, её ещё называют ЭТИМ по-русски.
Ответ: ривер

**Хорошо:**
Вопрос: В покере — пятая общая карта на столе.
Ответ: ривер

**Текст:**
апельсиновая сделка — СССР и Израиль — отсюда Чебурашка приехал

**Плохо:**
Вопрос: Советско-израильскую торговую сделку, связанную с апельсинами, называют ЭТИМ.
Ответ: апельсиновая сделка
Почему плохо: и слово "сделка", и однокоренное слово "апельсин" есть в вопросе.

**Хорошо:**
Вопрос: Известное торговое соглашение между СССР и Израилем.   
Ответ: апельсиновая сделка

## Пример 6

**Правило:**
Если спрашивается только год, а не вся дата, то вопрос и должен быть В ЭТОМ ГОДУ.

**Плохо:**
Вопрос: Идея Velcro пришла де Местралю В ЭТУ ДАТУ.
Ответ: 1941

**Хорошо:**
Вопрос: Идея Velcro пришла Жоржу де Местралю В ЭТОМ ГОДУ.
Ответ: 1941

## Пример 7

**Правило:**
Написание терминов / имён итд по-русски берём из _русской_ (НЕ английской) википедии.

**Плохо:**
Ответ: Огюст Дюпин

**Хорошо:**
Ответ: Огюст Дюпен

## Пример 8

**Правило:**
В текстах вопросов и ответов указывать не только фамилию, но и имя (если есть и имя, и фамилия).

**Плохо:**
Вопрос: Чабб купил Стоунхэндж на аукционе В ЭТОМ ГОДУ.

**Хорошо:**
Вопрос: Сесил Чабб купил Стоунхэндж на аукционе В ЭТОМ ГОДУ.

## Пример 9

**Правило:**
Падеж, род и число должны быть согласован и в словах-заменах (ОН, ОНА, ОНО итд).

**Плохо:**
Фил Найт основал Nike вместе с бывшим тренером по лёгкой атлетике — ОН.

**Хорошо:**
Фил Найт основал Nike вместе с бывшим тренером по лёгкой атлетике — ИМ.


## Пример 10
Не повторять даже часть слов ответа в вопросе.

**Плохо:**
Вопрос: Эффект Флинна назван в честь ЭТОГО психолога.
Ответ: Джеймс Флинн

**Хорошо:**
Вопрос: Эффект постепенного роста результатов IQ-тестов с течением времени называют в честь ЭТОГО психолога.
Ответ: Джеймс Флинн

**Плохо:**
Вопрос: Постепенный рост результатов IQ-тестов с течением лет называют ЭТИМ эффектом.
Ответ: эффект Флинна

**Хорошо:**
Вопрос: Постепенный рост результатов IQ-тестов с течением лет называют ИМ.
Ответ: эффект Флинна

## Пример 11
Если в вопросе спрашивается число (которое не является годом), то вопрос будет в форме СТОЛЬКО. Число может включать в себя единицу измерения.

**Плохо:**
Вопрос: Длина маршрута Крымского троллейбуса составляет примерно ЭТО километров.
Ответ: 86

**Лучше:**
Длина маршрута Крымского троллейбуса составляет примерно СТОЛЬКО километров.
Ответ: 86

**Хорошо:**
Длина маршрута Крымского троллейбуса составляет примерно СТОЛЬКО.
Ответ: 86 километров

## Пример 12
Для вопроса "как звали ЕГО/ЕЁ" нужно использовать форму "звали ТАК", а не "звали ОН" / "звали ОНА"

**Плохо:**
Вопрос: Льюиса Кэрролла по-настоящему звали ОН.
Ответ: Чарльз Лютвидж Доджсон

**Хорошо:**
Вопрос: Льюиса Кэрролла по-настоящему звали ТАК.
Ответ: Чарльз Лютвидж Доджсон

## Пример 13
Если в вопросе просто спрашивается определение реалии, то не нужно подставлять вопросное слово, достаточно просто определения.
В этом случае вопрос задаётся в именительном падеже.

**Плохо:**
Вопрос: Певца, растерзанного менадами, звали — ОН.
Ответ: Орфей

**Хорошо:**
Вопрос: Певец, растерзанный менадами.
Ответ: Орфей

**Плохо:**
Английское «Yes, sir» в русской военной среде часто заменяют одним словом — ЭТИМ.

**Хорошо:**
Английское «Yes, sir» в русской военной среде превратилось в ЭТО СЛОВО.


## Пример 14
Не "прозвище ЭТО", а "ТАКОЕ прозвище" или "ЭТО прозвище"

**Плохо:**
Вопрос: Немецкий клуб, почти 55 лет непрерывно игравший в высшем дивизионе, носит прозвище ЭТО.

**Хорошо:**
Вопрос: Гамбург, почти 55 лет непрерывно игравший в высшем дивизионе, носит ЭТО прозвище.

**Хорошо:**
Вопрос: Гамбург, почти 55 лет непрерывно игравший в высшем дивизионе, носит ТАКОЕ прозвище.

## Пример 15
Не нужно заменять остальные реалии из текста кроме той, которая спрашивается в ответе именно этой карточки.

Текст:
ФК Гамбург — прозвище Динозавры (очень долго находились в высшей лиге, 55 лет подряд)

**Плохо:**
Вопрос: Немецкий клуб, почти 55 лет непрерывно игравший в высшем дивизионе, носит ЭТО прозвище.
Ответ: Динозавры

**Хорошо:**
Вопрос: ФК Гамбург, почти 55 лет непрерывно игравший в высшем дивизионе, носит ЭТО прозвище.
Ответ: Динозавры

## Пример 16
Нельзя иметь однокоренные слова в вопросе и ответе, даже если это имя собственное.

**Плохо:**
Вопрос: Эффект Рингельмана впервые описал ОН.
Ответ: Максимилиан Рингельманн

**Хорошо:**
Вопрос: Эффект снижения личной продуктивности в большой группе описал ОН.
Ответ: Максимилиан Рингельманн

**Плохо:**
Вопрос: Пакетики против протечки ручки в советском гражданском авиапарке раздавали пассажирам ТАМ.
Ответ: в ранних советских самолётах

**Хорошо:**
Вопрос: Пакетики против протечки ручки раздавали ТАМ.
Ответ: в ранних советских самолётах

# Инструкция: генерация карточек из Telegram

Карточки генерирует **LLM-агент** (Cursor), не Python-эвристики.

## Референсы (прочитать перед генерацией)

1. [`hermes/quizlet-rules.md`](../../hermes/quizlet-rules.md) — правила и примеры хорошо/плохо
2. [`parsing-examples/parsing-examples.json`](../../parsing-examples/parsing-examples.json) — 7 golden examples
3. [`docs/text_parsing_guide.md`](../../docs/text_parsing_guide.md) — типы entities и формы замен
4. [`style_samples.json`](style_samples.json) — стиль вопросов из quizlet-modules

## Вход / выход

- Вход: `batches/batch-NNN.json` (5 сообщений)
- Выход: `generations/batch-NNN.json` — массив examples в том же формате + `message_id`

После записи:

```bash
python cursor/cards-llm/log_llm_call.py --id gen-batch-001 --operation generate_cards \
  --batch batch-001.json --messages 5 --generation batch-001.json \
  --tool-calls '[{"tool":"WebFetch","url":"...","purpose":"..."}]'
```

## Алгоритм на каждое сообщение

1. Определи тип: `text-only` / `link-only` / `link+comment` / `question`
2. Выдели `entities`: `{name, type}` (type из guide или `""`)
3. Факты:
   - есть ссылка → прочитай страницу (приоритет ru.wikipedia.org)
   - неполные имена → дополни из ru.wikipedia
   - текст — вопрос → найди ответ, переформулируй вопрос (ОН/ТАК, не «Кто»)
4. Сгенерируй 1–4 карточки на интересные реалии
5. Каждая карточка: `question`, `answer`, `logic`
6. Самопроверка по quizlet-rules

## Чеклист (включая правила 13–16)

- [ ] Карточка на каждую entity из списка
- [ ] В вопросе сохранены реалии из исходного текста, не заменены синонимами (пример 15)
- [ ] «ЭТО прозвище» / «ТАКОЕ прозвище», не «прозвище ЭТО» (пример 14)
- [ ] «превратилось в ЭТО СЛОВО», не «ЭТИМ» (пример 13)
- [ ] Нет однокоренных слов в вопросе и ответе **одной пары** (пример 16); в разных карточках одного сообщения пересечения допустимы
- [ ] Вопрос однозначен без других карточек: называй эпизод/книгу/персонажа по имени, не «этой серии» / «эпизода»
- [ ] Нет «Кто», «Когда», «Сколько» в вопросе
- [ ] Персоналии → ОН/ОНА/ИМ/ЕЙ (согласование)
- [ ] Год → В ЭТОМ ГОДУ; дата → В ЭТУ ДАТУ; число → СТОЛЬКО
- [ ] Официальные названия → вопрос с ТАК, ответ полный
- [ ] «звали ТАК», не «звали ОН»
- [ ] Нет слов/корней ответа в вопросе этой пары
- [ ] Ответ ≤ 12 слов
- [ ] Имена из русской Wikipedia, полное имя+фамилия
- [ ] Английский ответ → язык объявлен в вопросе

## Формат выхода (один example)

```json
{
  "message_id": -999900123,
  "id": "slug-from-text",
  "text": "исходный текст сообщения",
  "tags": ["text-only", "multi-card"],
  "entities": [
    {"name": "Иосиф Бродский", "type": "PERSONALIA_MALE"}
  ],
  "cards": [
    {
      "question": "ОН сказал: «...»",
      "answer": "Иосиф Бродский",
      "logic": "..."
    }
  ]
}
```

`message_id` обязателен в generations; в финальном `parsing-examples-cursor.json` его нет.
